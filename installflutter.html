HOW TO INSTALL FLUTTER

  1. make a directory called Android
  1. unzip this folder and rename it openjdk https://github.com/AdoptOpenJDK/openjdk8-binaries/releases/download/jdk8u202-b08/OpenJDK8U-jdk_x64_windows_hotspot_8u202b08.zip
  2. download this https://developer.android.com/studio/#command-tools and then put it like Android\cmdline-tools\tools

  1. download flutter sdk or clone it in src\ folder from https://flutter.io/docs/get-started/install/windows

  1. add the following environment variables
  1. JAVA_HOME “C:\Android\openjdk”
  2. ANDROID_HOME “C:\Android”
  3. ANDROID_SDK_ROOT “C:\Android\cmdline-tools”

  1. to %path% add
  1. “C:\Android\cmdline-tools”
  2. "C:\Android\cmdline-tools\tools\bin"
  3. "C:\src\flutter\bin"

  1. copy the contents of the:

...cmdline-tools\tools\Lib\_ to cmdline-tools\tools\Lib\

  1. Check for sdkmanager --list and then download the correct versions of the following:
  1. sdkmanager “system-images;android-27;default;x86_64”
  2. sdkmanager “platform-tools”
  3. sdkmanager “build-tools;27.0.3”
  4. sdkmanager “platforms;android-27”
  5. sdkmanager emulator (optional)

  1. DOWNLOAD VS-code and set it up for flutter and dart

  1. a path to code.exe as a shortcut or the default installation path should be present in

C:\Users\<user>\AppData\Local\Programs\Microsoft VS Code

  1. Run "flutter doctor" in windows cmd

SETTING UP A DEVICE

  1. To set up a wireless debugging device
  1. add the working directory of adb.exe to your %path% present in C:\ANDROID\platform-tools (this will be available after you install platform tools with the sdkmanager)
  2. adb devices
  3. adb tcpip 1234 (you can give any 4 digit port)
  4. adb connect [ip address to which mobile is connected]:{port}
  5. adb devices (to confirm that the device is connected wirelessly)
